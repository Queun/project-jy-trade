# Excel 字段字典

本文件记录当前 Excel 输入、输出和业务字段契约。系统按已确认的 Sheet 和列名读取、导出，不做列名猜测；代码规则变化时必须同步更新本文和导出测试。

## 3. 确定发货单

文件：`ole案例文件——发货前\3确定发货单\待推单.xlsx`

核心 Sheet：`订货通知单明细`

字段：

| 字段 | 用途 |
| --- | --- |
| 订货单号 | 确定发货明细归属 |
| 通知单号 | 上游通知单号 |
| 收货方编码 | 门店编码 |
| 收货方名称 | 门店名称 |
| 业务员 | 原始业务员 |
| 截止日期 | 发货参考 |
| 商品编码 | 商品编码 |
| 商品条码 | 商品条码 |
| 商品名称 | 商品名称 |
| 规格 | 商品规格 |
| 订货数量 | 原始订货量 |
| 发货数量 | 确定单“实际发货数量”；导入后保存为 `plannedShipQty`，作为本轮自动分配的需求上限 |
| 合同进价 | 价格参考 |
| 主仓 | 上游历史分仓字段；仅保留在原始行 JSON 中追溯，不参与系统建议或最终仓库选择 |
| 临期仓 | 上游历史分仓字段；仅保留在原始行 JSON 中追溯，不参与系统建议或最终仓库选择 |
| 备注 | 做单或仓库处理备注 |

确定单导入后的审核数量模型：

| 系统字段 | 页面含义 | 来源与约束 |
| --- | --- | --- |
| `orderQty` | 订货 | 确定单 `订货数量`；只用于展示和追溯 |
| `plannedShipQty` | 发货 | 确定单 `实际发货数量`；自动建议不得超过此值 |
| `suggestedShipQty` | 系统建议 | 按最新成功的本地可发库存快照、VIP 优先、同级公平分配和单行单仓规则计算 |
| `approvedShipQty` | 最终发货数量 | 用户确认值；可高于建议或计划，但页面提示风险，导出读取此值 |

补充规则：

- `实际发货数量` 为 0 的行仍进入审核，建议量和最终量默认为 0，不占用库存，也不要求仓库；用户可以人工改为正数。
- 系统建议同时给出 `suggestedWarehouseNo` / `suggestedWarehouseName`，审核决定使用独立的 `fulfillmentWarehouseNo` / `fulfillmentWarehouseName`。
- 库存查询失败时建议数量为 0、建议仓库为空；技术错误保存在 `stockErrorDetail`，人工填写正数并选择仓库后可在二次确认下提交。
- 未匹配商品行不会静默丢弃。最终数量为 0 时可提交；正数时必须先形成有效商品映射并选择已启用仓库。
- 确定单导入和每次重新校验后都处于待审核状态；只有提交审核完成后才能做单导出。
- 批次通过 `stockSnapshotRunId` / `stockSnapshotAt` 记录建议所依据的库存快照；导出不会因后台产生新快照而改变，只有用户主动重新校验才采用新库存。

## 4. 旺店通批量做单模板

参考模板：`ole案例文件——发货前\4批量做单表格\批量导入模板.xls`

Sheet：`Sheet1`（做单表）、`不做单表`（核对表）

当前系统按模板固定输出 49 列，文件格式为 OOXML `.xlsx`。首个 `Sheet1` 保持旺店通导入模板字段兼容，只包含最终发货数量大于 0 的做单行；`不做单表` 使用完全相同的 49 列，包含所有最终发货数量为 0 的行，供人工核对。两个 Sheet 均覆盖完整使用区域启用自动筛选，文件打开时仅激活 `Sheet1`。核心字段：

| 字段 | 用途 |
| --- | --- |
| 店铺名称 | 固定或配置 |
| 原始单号 | 系统按“门店 + 最终仓库”分组生成 16 位编号，格式为 `JY` + 6 位日期 + 8 位批次/组序号字符；同组商品行重复使用同一编号 |
| 收件人 | 地址匹配表派生 |
| 省 / 市 / 区 | 地址拆分或留空 |
| 手机 / 固话 | 地址匹配表派生 |
| 仓库 | 配置或审核结果 |
| 地址 | 地址匹配表派生 |
| 物流类别 | 固定或配置 |
| 仓库名称 | 审核决定中的 `fulfillmentWarehouseName`，不在导出时重新分配或写死默认仓 |
| 物流公司 | 做单物流 |
| 买家备注 / 客服备注 / 打印备注 | `客服备注` 按同一门店汇总本批次通知单号并用顿号分隔；其他备注按已确认业务规则填充 |
| 发票类型 / 发票抬头 | 当前案例中存在默认值 |
| 业务员 | 当前执行导出的登录账号用户名 |
| 商家编码 | `wdtMakeOrderCode`；普通商品通常为 `spec_no`，组合装为 `suite_no`，缺少做单码时回退到 `spec_no` |
| 货品数量 | 最终审核发货数量 `approvedShipQty` |
| 货品价格 / 货品总价 / 货品优惠 | 按做单规则填充 |
| 平台 | 固定或配置 |

当前导出规则：

- `Sheet1` 只导出 `approvedShipQty > 0` 的行；正数最终数量在保存审核决定时统一归一为发货。
- 同一文件的 `不做单表` 收录全部 `approvedShipQty = 0` 的行，不依赖 `decision` 是待处理、发货还是不发货，避免库存不足建议、未匹配行或手工归零行静默遗漏。
- `不做单表` 与 `Sheet1` 使用相同字段生成规则，不增加原因或状态等额外列；最终仓库、商家编码等未填写内容保持为空，货品数量按最终审核数量输出为 0。
- `原始单号` 是旺店通合并键和仓库派单标识，不要求逐行唯一，也不直接复用通知单号。
- 同一门店、同一最终仓库的所有商品行使用相同 `原始单号`；同一门店发往不同仓库时拆成不同编号，不同门店也始终使用不同编号。
- 相同批次和审核结果重复导出时，分组编号保持稳定。
- 同一门店的多个通知单号汇总到 `客服备注`，用于仓库和客服回查。
- 做单导出只读取已审核的 `approvedShipQty` 和最终仓库，不重新查询库存或调整分仓。
- 普通商品库存核对和做单通常都使用 `spec_no`。
- 单组件组合装用组件 `spec_no` 核对库存，用组合装 `suite_no` 做单。
- 49 列完整顺序由 API 导出测试保护；后续仍需用客户最终模板做样例级比对。

## 5. 地址匹配表

文件：`ole案例文件——发货前\4批量做单表格\地址匹配表格.xlsx`

当前文件包含多个历史 Sheet，表头不完全统一。优先使用结构最稳定的字段：

- 门店编号/群组。
- 门店名称。
- 门店地址。
- 收货人。
- 联系方式。
- VIP 标记：系统内人工维护字段，导入地址时保留已有标记，不从历史 Excel 自动推断；库存初审时用于优先分货。

后续需要确定：

- 以哪个 Sheet 作为当前有效地址源。
- 门店编码是否唯一。
- 若同一门店多条地址，使用最新表、指定表，还是人工选择。

## 6. 小样和套盒维护表

当前样例：

- `ole-中小样价格统计26.6.22(1)(1).xls`
- `ole新套盒(1).xlsx`

这些文件来自人工统计，字段口径先作为维护数据导入基线，不直接替代订货单字段。

### 小样价格

Sheet：`小样价格`

当前读取字段：

| 字段 | 用途 |
| --- | --- |
| 商品编码 | 客户侧商品编码，可作为组件编号候选 |
| 商品条码 | 优先作为组件编号匹配本地 WDT 商品档案 |
| 商品全称 / 商品名称 | 外部商品名称，匹配失败时用于人工识别 |
| 标签价格 | 价格参考，写入备注 |
| 系统供货价 | 价格参考，写入备注 |

当前导入策略：

- 每行小样作为一个 `sample` 外部维护商品。
- 小样自身也是一个主组件，优先用商品条码匹配本地 WDT 商品档案；无条码时再用商品编码。
- 唯一命中未删除 WDT 规格时标记 `confirmed`，否则标记 `needs_review`。
- 保留源 Sheet、源行号和原始字段，便于回查人工表。

### 套盒

Sheet：`套盒`

当前读取字段：

| 字段 | 用途 |
| --- | --- |
| 品名 | 套盒外部商品名称 |
| 规格 | 原始规格备注 |
| 套盒条码 | 套盒外部条码 |
| 组件编码1 | 主组件编号，优先匹配 WDT 规格 |
| 组件备注1 | 主组件数量或人工备注 |
| 合同价 | 套盒价格参考，写入备注 |
| 组件编码2 | 替换组件编号，先进入人工确认 |
| 组件备注2 | 替换组件数量或人工备注 |
| 组件名称2 | 替换组件名称 |
| 组件价格2 | 替换组件价格参考 |
| 组件编码3 | 赠品/额外组件编号，先进入人工确认 |
| 组件备注3 | 赠品/额外组件数量或人工备注 |

当前导入策略：

- 一个套盒条码或套盒名称对应一个 `bundle` 外部维护商品。
- 主组件可自动匹配本地 WDT 商品档案；替换组件和额外组件先强制 `needs_review`，避免误把人工备注当成确定发货规则。
- 组件备注中的数字会暂作为数量倍数读取；后续仍需要人工确认真实业务含义。

### 联营套盒

Sheet：`联营套盒`

当前读取字段按位置整理：

| 字段 | 用途 |
| --- | --- |
| 套盒名称 | 套盒外部商品名称 |
| 商品条码 | 组件编号候选 |
| 商品名称 | 组件名称 |
| 规格 | 组件规格 |
| 原价 | 价格参考 |
| 特售价 | 价格参考 |
| 数量备注 | 组件数量或人工备注 |
| 备注1-备注4 | 人工备注，保留原始信息 |

待确认：

- 套盒条码、组件条码、商品编码和 WDT `spec_no`/`goods_no` 的具体边界。
- 替换组件、赠品和备注组件是否参与库存扣减，还是仅用于审核提示。
- 同一外部编号在不同人工表中重复出现时，以最新导入、人工确认状态还是指定表为准。
