# Excel 字段字典

本文件记录当前样例 Excel 的字段基线。字段固定时，系统应按这些列名读取和导出，不做列名猜测。

## 1. 订货单

| 送货方式 | 原始配送方式 |

| 送货地 | 门店/送货地显示值 |
| 大类 | 商品大类 |
| 订货日期 | 订单排序字段 |
| 截止日期 | 审核参考字段 |
| 上传时间 | 优先订单排序字段 |
| 业务员 | 审核参考字段 |

| 制单时间 | 原始字段保留 |
| 审核人 | 原始字段保留 |

- 重复条码数：2 个。

## 5. 地址匹配表

文件：`ole案例文件——发货前\4批量做单表格\地址匹配表格.xlsx`

当前文件包含多个历史 Sheet，表头不完全统一。优先使用结构最稳定的字段：

- 门店编号/群组。
- 门店名称。
- 门店地址。
- 收货人。
- 联系方式。
- VIP 标记：系统内人工维护字段，导入地址时保留已有标记，不从历史 Excel 自动推断；库存初审时用于优先分货。

后续需要确定：

- 以哪个 Sheet 作为当前有效地址源。
- 门店编码是否唯一。
- 若同一门店多条地址，使用最新表、指定表，还是人工选择。

## 6. 小样和套盒维护表

当前样例：

- `ole-中小样价格统计26.6.22(1)(1).xls`
- `ole新套盒(1).xlsx`

这些文件来自人工统计，字段口径先作为维护数据导入基线，不直接替代订货单字段。

### 小样价格

Sheet：`小样价格`

当前读取字段：

| 字段 | 用途 |
| --- | --- |
| 商品编码 | 客户侧商品编码，可作为组件编号候选 |
| 商品条码 | 优先作为组件编号匹配本地 WDT 商品档案 |
| 商品全称 / 商品名称 | 外部商品名称，匹配失败时用于人工识别 |
| 标签价格 | 价格参考，写入备注 |
| 系统供货价 | 价格参考，写入备注 |

当前导入策略：

- 每行小样作为一个 `sample` 外部维护商品。
- 小样自身也是一个主组件，优先用商品条码匹配本地 WDT 商品档案；无条码时再用商品编码。
- 唯一命中未删除 WDT 规格时标记 `confirmed`，否则标记 `needs_review`。
- 保留源 Sheet、源行号和原始字段，便于回查人工表。

### 套盒

Sheet：`套盒`

当前读取字段：

| 字段 | 用途 |
| --- | --- |
| 品名 | 套盒外部商品名称 |
| 规格 | 原始规格备注 |
| 套盒条码 | 套盒外部条码 |
| 组件编码1 | 主组件编号，优先匹配 WDT 规格 |
| 组件备注1 | 主组件数量或人工备注 |
| 合同价 | 套盒价格参考，写入备注 |
| 组件编码2 | 替换组件编号，先进入人工确认 |
| 组件备注2 | 替换组件数量或人工备注 |
| 组件名称2 | 替换组件名称 |
| 组件价格2 | 替换组件价格参考 |
| 组件编码3 | 赠品/额外组件编号，先进入人工确认 |
| 组件备注3 | 赠品/额外组件数量或人工备注 |

当前导入策略：

- 一个套盒条码或套盒名称对应一个 `bundle` 外部维护商品。
- 主组件可自动匹配本地 WDT 商品档案；替换组件和额外组件先强制 `needs_review`，避免误把人工备注当成确定发货规则。
- 组件备注中的数字会暂作为数量倍数读取；后续仍需要人工确认真实业务含义。

### 联营套盒

Sheet：`联营套盒`

当前读取字段按位置整理：

| 字段 | 用途 |
| --- | --- |
| 套盒名称 | 套盒外部商品名称 |
| 商品条码 | 组件编号候选 |
| 商品名称 | 组件名称 |
| 规格 | 组件规格 |
| 原价 | 价格参考 |
| 特售价 | 价格参考 |
| 数量备注 | 组件数量或人工备注 |
| 备注1-备注4 | 人工备注，保留原始信息 |

待确认：

- 套盒条码、组件条码、商品编码和 WDT `spec_no`/`goods_no` 的具体边界。
- 替换组件、赠品和备注组件是否参与库存扣减，还是仅用于审核提示。
- 同一外部编号在不同人工表中重复出现时，以最新导入、人工确认状态还是指定表为准。
