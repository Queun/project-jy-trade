# 项目实施计划

## 目标流程

系统先不替代甲方现有 Excel 做单习惯，先把人工流程中最容易出错、最耗时的库存核对和发货初审标准化。

目标流程：

```text
导入订货单
  -> 商品匹配
  -> 查询旺店通库存
  -> 生成订货初审列表
  -> 审核人确认发货/不发货/调整数量
  -> 生成确定发货单
  -> 导出做单 Excel
```

## 阶段一：样表和字段基线

产出：

- 订货单字段字典。
- 订货初审单字段字典。
- 确定发货单字段字典。
- 旺店通批量做单模板字段字典。
- 地址匹配表字段字典。

重点：

- 字段固定，不做“自动猜列名”。
- 保留原始 Excel 行号，方便回查。
- 先识别必须字段、派生字段、审核人填写字段、导出字段。

## 阶段二：旺店通测试环境验证

产出：

- 接口签名和请求封装。
- 仓库查询返回样例。
- 货品/规格查询返回样例。
- 库存查询返回样例。
- 字段映射表。

重点：

- 只使用测试环境和只读接口。
- 不在仓库中保存任何真实密钥。
- 确认测试环境与正式环境的差异，尤其是真实店铺、仓库、库存数据不可用时的替代验证方式。

## 阶段三：商品匹配规则

产出：

- 外部订货单商品到旺店通 `spec_no` 的匹配规则。
- 匹配失败、多个候选、名称不一致的处理规则。
- 可维护的人工映射表。
- 小样、套装、赠品等外部维护商品表。

建议规则：

- 公开条码命中唯一规格时自动匹配。
- 条码命中多个规格时进入人工确认。
- 条码未命中但名称高度相似时只提示候选，不自动确认。
- 小样、套装、赠品优先依赖人工维护映射，不硬编码在程序里。

当前进展：

- 已新增外部维护商品数据模型：`external_products` 和 `external_product_components`，独立于现有一对一 `product_mappings`，避免影响当前订单初审主链路。
- 已新增小样/套盒 Excel 导入预览和确认导入 API，可保留源文件名、Sheet、源行号、原始字段、导入人和导入时间。
- 导入预览只读取本地旺店通商品档案缓存做编号匹配，不调用旺店通写接口，不接入真实库存扣减。
- 小样按商品条码/商品编码作为单组件维护；套盒按一个外部套盒对应多个组件维护。
- 套盒主组件可在唯一命中 WDT 规格时自动确认；替换组件、赠品或备注组件先强制 `needs_review`，避免把人工备注误当成确定映射。

下一步：

- 增加人工修正能力：组件改绑 WDT 规格、禁用维护项、确认替换/赠品组件。
- 设计但暂不立即接入库存初审的匹配顺序：订货单编号直接命中 WDT 规格 -> 外部维护商品表 -> 现有人工一对一映射 -> 名称候选提示。
- 接入初审前需要再做一轮真实样例核对，明确外部商品编号、商品条码、套盒条码和 WDT `spec_no`/`goods_no` 的映射边界。

## 阶段四：库存初审逻辑

产出：

- 库存快照表。
- 按订货时间排序的滚动库存分配算法。
- 初审状态：库存充足、库存不足、需评估、未匹配。
- 审核页面或审核表格。

重点：

- 同一批订单中重复商品要按订货先后顺序扣减可发库存。
- 系统只给建议，最终发不发由审核人决定。
- 审核人可以选择不发、改发货数量、填写原因。

## 阶段五：确定发货和做单 Excel

产出：

- 确定发货单导出。
- 旺店通批量做单 Excel 导出。
- 单据凭证留存。
- 本批次处理日志。
- 当前已完成三类 `.xlsx` 基础导出：初审单、确定发货单、做单 Excel。
- 当前已完成导出记录持久化、导出历史展示和下载接口。

重点：

- 暂不直接推送旺店通或仓库。
- 导出格式优先贴合现有 Excel 模板。
- 每次导出保留批次号、原订货单文件名、处理时间、审核人。
- 下一步需要按客户最终模板细化字段顺序、样式、合并单元格和凭证留存要求。

## 阶段六：最小系统形态

建议 MVP 包含：

- 简单登录。
- 订货单上传。
- 库存查询和匹配状态展示。
- 审核勾选、数量调整、原因填写。
- 确定发货单导出。
- 做单 Excel 导出。
- 批次历史记录。

暂缓：

- 正式环境写入。
- 自动推送仓库。
- 复杂权限体系。
- 复杂工作流审批。

当前进展：

- 已完成用户名密码登录、session cookie、退出登录。
- 已完成业务接口登录保护。
- 已完成审核工作台、批次历史、导出历史和下载入口。
- 已完成正式订单上传、商品档案同步、真实初审、地址维护和管理员删除批次。
- 已完成小样/套盒维护数据的后端模型、导入预览、确认导入、前端维护入口和测试，尚未接入库存初审流程。
- 用户管理、角色权限细分和复杂审批暂缓。

## 后续质量项

### 导出字段字典和样例比对测试

背景：

- 初审单、确定发货单和做单表格已经具备导出能力。
- 后续业务字段仍会调整，暂时不在当前清理轮强行锁死字段。
- 做单表格是核心产物，需要建立更硬的回归保护。

后续单独开发：

- 为三类导出建立字段对照字典，记录每个字段来自原始订单、审核结果、门店地址、商品档案、库存查询、人工维护还是暂空。
- 明确哪些字段必须填、哪些字段可暂空、哪些字段是业务推导。
- 建立样例级导出比对测试，至少验证表头、关键列顺序、关键字段值、空字段策略和文件可打开。
- 做单表格优先级最高，初审明细和确定发货单次之。
