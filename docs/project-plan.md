# 项目实施计划

## 目标流程

系统先不替代甲方现有 Excel 做单习惯，先把人工流程中最容易出错、最耗时的库存核对和发货初审标准化。

目标流程：

```text
导入订货单
  -> 商品匹配
  -> 查询旺店通库存
  -> 生成订货初审列表
  -> 审核人确认发货/不发货/调整数量
  -> 生成确定发货单
  -> 导出做单 Excel
```

## 阶段一：样表和字段基线

产出：

- 订货单字段字典。
- 订货初审单字段字典。
- 确定发货单字段字典。
- 旺店通批量做单模板字段字典。
- 地址匹配表字段字典。

重点：

- 字段固定，不做“自动猜列名”。
- 保留原始 Excel 行号，方便回查。
- 先识别必须字段、派生字段、审核人填写字段、导出字段。

## 阶段二：旺店通测试环境验证

产出：

- 接口签名和请求封装。
- 仓库查询返回样例。
- 货品/规格查询返回样例。
- 库存查询返回样例。
- 字段映射表。

重点：

- 只使用测试环境和只读接口。
- 不在仓库中保存任何真实密钥。
- 确认测试环境与正式环境的差异，尤其是真实店铺、仓库、库存数据不可用时的替代验证方式。

## 阶段三：商品匹配规则

产出：

- 外部订货单商品到旺店通 `spec_no` 的匹配规则。
- 匹配失败、多个候选、名称不一致的处理规则。
- 可维护的人工映射表。

建议规则：

- 公开条码命中唯一规格时自动匹配。
- 条码命中多个规格时进入人工确认。
- 条码未命中但名称高度相似时只提示候选，不自动确认。
- 小样、套装、赠品优先依赖人工维护映射，不硬编码在程序里。

## 阶段四：库存初审逻辑

产出：

- 库存快照表。
- 按订货时间排序的滚动库存分配算法。
- 初审状态：库存充足、库存不足、需评估、未匹配。
- 审核页面或审核表格。

重点：

- 同一批订单中重复商品要按订货先后顺序扣减可发库存。
- 系统只给建议，最终发不发由审核人决定。
- 审核人可以选择不发、改发货数量、填写原因。

## 阶段五：确定发货和做单 Excel

产出：

- 确定发货单导出。
- 旺店通批量做单 Excel 导出。
- 单据凭证留存。
- 本批次处理日志。
- 当前已完成三类 `.xlsx` 基础导出：初审单、确定发货单、做单 Excel。
- 当前已完成导出记录持久化、导出历史展示和下载接口。

重点：

- 暂不直接推送旺店通或仓库。
- 导出格式优先贴合现有 Excel 模板。
- 每次导出保留批次号、原订货单文件名、处理时间、审核人。
- 下一步需要按客户最终模板细化字段顺序、样式、合并单元格和凭证留存要求。

## 阶段六：最小系统形态

建议 MVP 包含：

- 简单登录。
- 订货单上传。
- 库存查询和匹配状态展示。
- 审核勾选、数量调整、原因填写。
- 确定发货单导出。
- 做单 Excel 导出。
- 批次历史记录。

暂缓：

- 正式环境写入。
- 自动推送仓库。
- 复杂权限体系。
- 复杂工作流审批。

当前进展：

- 已完成用户名密码登录、session cookie、退出登录。
- 已完成业务接口登录保护。
- 已完成审核工作台、批次历史、导出历史和下载入口。
- 用户管理、角色权限细分和正式上传订货单仍在后续阶段。
